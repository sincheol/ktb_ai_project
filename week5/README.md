# 인공지능 프로젝트 결과 보고서

프로젝트 주제: 뇌졸중(Stroke) 발병 위험 예측을 위한 머신러닝 모델 비교 분석

## 1. 서론 (Introduction)

### 1.1 프로젝트 배경 및 필요성

뇌졸중은 전 세계적으로 주요 사망 원인 중 하나이며, 발병 시 심각한 신체적 장애를 초래할 수 있다. 조기 진단과 위험 요인 관리는 뇌졸중 예방에 필수적이다. 의료 데이터의 축적과 인공지능 기술의 발전으로 환자의 임상 데이터를 분석하여 발병 가능성을 사전에 예측하는 것이 가능해졌다. 본 프로젝트에서는 환자의 생체 정보 및 생활 습관 데이터를 활용하여 뇌졸중 발병 여부를 예측하는 다양한 머신러닝 및 딥러닝 모델을 구축하고, 그 성능을 비교 분석하여 의료 보조 도구로서의 가능성을 탐색하고자 한다.

### 1.2 프로젝트 목표

본 프로젝트의 목표는 환자의 인구통계학적 정보와 건강 검진 데이터를 기반으로 뇌졸중 발병 여부(0: 정상, 1: 뇌졸중)를 이진 분류하는 최적의 모델을 찾는 것이다. 이를 위해 KNN, Perceptron, Linear SVM, Random Forest, Naive Bayes 등 다양한 알고리즘을 적용하고, 각 모델의 정확도뿐만 아니라 재현율(Recall)과 정밀도(Precision)를 종합적으로 평가하여, 특히 데이터 불균형 환경에서의 모델 성능 한계와 개선 방안을 도출하는 것을 목표로 한다.

## 2. 데이터셋 설명 (Dataset Description)

### 2.1 데이터 개요

본 실험에는 Kaggle의 Stroke Prediction Dataset을 사용하였다. 총 데이터 샘플 수는 5,110개이며, 각 샘플은 환자의 고유 ID를 제외한 11개의 입력 변수(Feature)와 1개의 타겟 변수(Target)로 구성되어 있다.

입력 변수: 성별(Gender), 나이(Age), 고혈압(Hypertension), 심장병(Heart Disease), 결혼 여부(Ever Married), 직업 유형(Work Type), 거주지 유형(Residence Type), 평균 혈당(Avg Glucose Level), 체질량지수(BMI), 흡연 상태(Smoking Status)

타겟 변수: Stroke (0: 정상, 1: 뇌졸중 발병)

### 2.2 데이터 전처리 (Preprocessing)

수집된 데이터의 품질을 높이고 모델 학습에 적합한 형태로 변환하기 위해 다음과 같은 전처리 과정을 수행하였다.

결측치 처리: 데이터 탐색 결과 'bmi' 변수에 약 201개의 결측치(NaN)가 존재함을 확인하였다. 해당 결측치는 전체 데이터 분포를 고려하여 중간값(Median)으로 대체(Imputation)하였다.

데이터 분할: 전체 데이터를 학습 및 검증용 데이터(Train/Validation)와 평가용 데이터(Test)로 7:3 비율로 분할하였으며, 클래스 불균형을 고려하여 층화 추출(Stratified Split) 방식을 적용하였다.

특성 스케일링 및 인코딩: 수치형 데이터(나이, 평균 혈당, BMI)에는 MinMaxScaler를 적용하여 범위를 0과 1 사이로 정규화하였고, 범주형 데이터(성별, 직업 등)에는 OneHotEncoder를 적용하여 수치 벡터로 변환하였다. 이 과정은 Scikit-learn의 ColumnTransformer와 Pipeline을 통해 자동화하였다.

## 3. 모델 설명 (Model Architecture)

본 프로젝트에서는 성능 비교를 위해 다음과 같은 다양한 머신러닝 알고리즘을 구현 및 실험하였다.

KNN (K-Nearest Neighbors): 데이터 간의 유클리드 거리를 계산하여 가장 가까운 K개의 이웃 정보를 기반으로 분류하는 직관적인 알고리즘이다. 본 실험에서는 PyTorch를 활용하여 사용자 정의 함수로 구현하였으며, K=5로 설정하여 실험하였다.

Perceptron: 가장 단순한 형태의 인공신경망으로, 입력 피처의 선형 결합을 통해 분류를 수행한다. 학습률(Learning Rate) 0.05, 에폭(Epoch) 50으로 설정하여 경사하강법을 통해 가중치를 학습시켰다.

Linear SVM (Support Vector Machine): 데이터 클래스를 가장 잘 구분하는 초평면(Hyperplane)을 찾는 알고리즘이다. Hinge Loss와 L2 정규화(Regularization)를 결합한 손실 함수를 정의하고, PyTorch를 사용하여 최적화 과정을 직접 구현하였다.

Random Forest: 다수의 결정 트리(Decision Tree)를 앙상블(Ensemble)하여 예측 성능과 안정성을 높인 모델이다. 100개의 트리를 사용하여 과적합을 방지하고 일반화 성능을 확보하고자 하였다.

Naive Bayes (GaussianNB): 특성들 간의 독립을 가정하고 베이즈 정리를 적용하는 확률 기반 분류기이다.

## 4. 실험 방법 (Experimental Setup)

### 4.1 실험 환경

실험은 Python 환경에서 진행되었으며, 데이터 처리를 위해 Pandas와 Scikit-learn 라이브러리를, 모델 구현 및 연산을 위해 PyTorch와 NumPy를 사용하였다.

### 4.2 평가지표

본 데이터셋은 뇌졸중 환자(Class 1)가 전체의 약 5% 미만인 극심한 불균형 데이터셋이다. 따라서 단순 정확도(Accuracy)만으로는 모델의 실질적인 성능을 평가하기 어렵다. 이에 따라 정밀도(Precision), 재현율(Recall), F1-score를 포함한 혼동 행렬(Confusion Matrix) 기반의 지표를 주요 평가 기준으로 삼았다. 특히, 실제 환자를 놓치지 않는 것이 중요한 의료 도메인의 특성상 재현율(Recall)에 중점을 두었다.

## 5. 결과 및 분석 (Results & Analysis)

### 5.1 정량적 결과

실험 결과, 대부분의 모델(KNN, Perceptron, SVM, Random Forest)이 약 **94~95%의 높은 정확도(Accuracy)**를 기록하였다. 그러나 세부 지표를 분석한 결과는 다음과 같았다.

다수 클래스(정상, 0): 정밀도와 재현율 모두 0.95 이상으로 매우 높게 나타났다.

소수 클래스(뇌졸중, 1): KNN, Perceptron, SVM 모델의 경우 재현율이 0.00~0.03 수준에 머물렀다. 이는 모델이 대부분의 샘플을 '정상'으로만 예측하여 높은 정확도를 얻었음을 의미하며, 실질적인 예측 능력은 매우 낮음을 시사한다.

Naive Bayes: 유일하게 소수 클래스에 대한 재현율이 높게 나타났으나, 그만큼 위양성(False Positive)이 증가하여 전체 정확도는 29% 수준으로 크게 하락하였다.

### 5.2 결과 분석

학습된 모델들이 뇌졸중 환자를 효과적으로 탐지하지 못하는 주된 원인은 데이터의 클래스 불균형(Class Imbalance) 문제에 있다. 전체 데이터 중 뇌졸중 케이스가 매우 적어, 모델이 소수 클래스의 패턴을 충분히 학습하지 못하고 다수 클래스로 편향되는 현상이 발생하였다. KNN 모델의 경우 K값이 2 이상일 때 소수 클래스를 전혀 예측하지 못하는 등 과적합 양상을 보였다.

## 6. 결론 (Conclusion)

### 6.1 요약 및 성과

본 프로젝트를 통해 뇌졸중 예측을 위한 데이터 전처리 파이프라인을 구축하고, 5가지 이상의 머신러닝 모델을 적용하여 성능을 비교하였다. 수치상으로는 95%의 정확도를 달성하였으나, 이는 데이터 불균형에 기인한 착시 현상임을 확인하였고, 의료 데이터 분석 시 정확도 외에 재현율과 F1-score 등 다각적인 평가가 필수적임을 입증하였다.

### 6.2 한계점 및 향후 과제

현재 모델은 실제 임상 현장에서 사용하기에는 뇌졸중 환자에 대한 감지 능력(Recall)이 현저히 부족하다. 이를 개선하기 위해 향후 연구에서는 다음과 같은 접근이 필요하다.

데이터 불균형 해소: SMOTE(Synthetic Minority Over-sampling Technique)와 같은 오버샘플링 기법을 적용하거나, 학습 시 소수 클래스에 더 큰 가중치를 부여하는(Class Weighting) 방법을 도입해야 한다.

모델 고도화: 단순 머신러닝 모델을 넘어, 더 깊은 층을 가진 딥러닝 모델을 적용하거나 데이터 증강(Data Augmentation) 기법을 탭러 데이터에 맞게 변형하여 적용해 볼 필요가 있다.
